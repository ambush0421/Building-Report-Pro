# Plan: Phase 2 Milestone - 서비스 고도화 및 확장

## 1. 개요 (Overview)
- **목적:** Phase 1(추천 엔진 및 비교 시스템)의 성공적인 구축을 바탕으로, 데이터의 깊이와 사용성을 대폭 강화하여 **'상용 수준의 완성도'**를 달성함.
- **핵심 키워드:**
  - **Deep Data:** 실거래가, 상권 등 심층 데이터 연동.
  - **Professional Output:** 고객 제출용 PDF 리포트 생성.
  - **Mobile First:** 현장 실무자를 위한 완벽한 모바일 경험.

## 2. 주요 목표 (Core Goals)

### 2.1 실거래가 심층 연동 (Real Trade Integration)
- **What:** 국토교통부 실거래가 API를 연동하여, 단순 호가(Asking Price)가 아닌 **'실제 거래된 가격'**을 비교표에 노출.
- **Why:** 의사결정권자는 "옆 건물 얼마에 팔렸어?"를 가장 궁금해함.
- **Detail:** 최근 3년 내 인근(반경 500m) 유사 물건 거래 사례 자동 매칭.

### 2.2 전문 PDF 리포트 생성 (PDF Export)
- **What:** 웹 화면을 그대로 캡처하는 수준을 넘어, A4 규격에 최적화된 **'제안서 스타일'**의 PDF 다운로드 기능.
- **Why:** 실무자가 대표님에게 보고할 때는 종이 문서나 PDF 파일이 필수적임.
- **Detail:** 표지, 목차, 비교 분석, 상세 스펙, 예상 견적서 순의 자동 페이지네이션.

### 2.3 모바일 현장 UX 최적화 (Mobile Optimization)
- **What:** 이동 중인 실무자가 스마트폰으로 빠르게 물건 정보를 확인하고 공유할 수 있는 UX.
- **Why:** 부동산 업무의 70%는 현장에서 이루어짐.
- **Detail:** 가로 스와이프 비교 카드, 지도 기반 빠른 탐색, 원터치 카카오톡 공유.

## 3. 우선순위 및 로드맵 (Roadmap)

| 순서 | 기능명 | 예상 기간 | 비고 |
|:---:|:---|:---:|:---|
| **1** | **실거래가 연동** | 2 Days | 데이터 정확도 확보가 핵심 |
| **2** | **PDF 생성 엔진** | 2 Days | `react-pdf` 또는 `puppeteer` 활용 |
| **3** | **모바일 UX 개편** | 1 Day | 반응형 레이아웃 튜닝 |

## 4. 기술 스택 확장
- **PDF:** `@react-pdf/renderer` (Client-side rendering)
- **Data:** 국토부 실거래가 Open API (New Key Required)
- **Map:** Kakao Map SDK (Clustering 강화)

## 5. 성공 기준 (Success Metrics)
- 실거래가 데이터 매칭 성공률 80% 이상.
- PDF 생성 시 레이아웃 깨짐 0건.
- 모바일 페이지 로딩 속도(LCP) 1.5초 이내.
