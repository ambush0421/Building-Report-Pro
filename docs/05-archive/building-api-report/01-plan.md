# Plan: 기업 이전용 사옥 견적 및 빌딩 보고서 시스템

## 1. 개요 (Overview)
- **목표:** 기업 이전을 고려하는 결정권자(CEO, 임원) 및 실무자에게 신속하고 정확한 부동산 정보를 제공하는 **반응형 웹/모바일 리포팅 툴** 구축.
- **핵심 가치:** 
  - **자동화:** 건축물대장 및 시세 기반 견적서/보고서 즉시 생성.
  - **접근성:** PC 웹 및 모바일 환경에서 완벽한 반응형 UX 제공.
  - **현대적 디자인:** 직관적이고 세련된 데이터 시각화 (Charts, Maps).
- **타겟 유저:** 
  - **1차:** 기업 이전 결정권자 (의사결정용 요약 리포트).
  - **2차:** 부동산 실무자/담당자 (상세 스펙 및 법적 리스크 확인).

## 2. 주요 기능 (Core Features)
1.  **기업 이전 맞춤형 보고서 (Corporate Relocation Report):**
    - 사옥 이전 시 필요한 예상 비용(보증금, 임대료, 관리비, 인테리어 비용 등) 산출.
    - 건물 매입 시 예상 수익률 및 금융 비용 분석 시뮬레이션.
2.  **건축물 데이터 자동화 (Building Data Automation):**
    - 국토교통부 건축물대장 API 연동 (표제부, 층별개요, 전유부).
    - 위반건축물, 용도지역, 주차대수 등 필수 체크리스트 자동 하이라이팅.
3.  **반응형 뷰어 및 공유 (Responsive Viewer):**
    - 모바일에서도 보기 편한 카드 뉴스 형태의 요약 뷰.
    - PC에서는 상세 분석이 가능한 대시보드 형태.
    - PDF 저장 및 링크 공유 기능.
4.  **시각화 (Visualization):**
    - 위치 기반 지도(Map) 연동.
    - 층별 면적 및 공실 현황 차트.

## 3. 기술 스택 (Tech Stack)
- **Core:** Node.js, Next.js (React Framework) - **Web & Mobile 대응**
- **UI/UX:** Tailwind CSS, Shadcn/UI (현대적 디자인)
- **CLI/Reporting (Optional):** Ink + React (서버 사이드 리포트 생성 또는 내부 관리자용 CLI 도구로 활용 가능성 열어둠)
- **Data:** 국토교통부 OpenAPI, Kakao/Naver Map API.

## 4. 데이터 모델 (Key Data Points)
- **Basic Info:** 대지면적, 연면적, 준공연도, 주차대수, 승강기.
- **Financial:** 평당 임대료/매매가, 보증금, 예상 관리비.
- **Legal:** 위반건축물 여부, 용도지역 지구, 건폐율/용적률 상한.

## 5. 성공 기준 (Success Metrics)
- 모바일 및 데스크톱에서 레이아웃 깨짐 없는 완벽한 반응형 구현.
- 주소 입력 후 보고서 생성까지 3초 이내 소요.
- 결정권자가 한눈에 파악 가능한 '요약(Executive Summary)' 섹션 구현.
