# Plan: Landing Web Value-up

## Goal
- 랜딩 페이지의 첫 인상과 전환력을 강화해 "무엇이 좋아지는지"를 더 빠르게 전달한다.

## Scope
- 랜딩 구성(`src/app/page.tsx`)에 가치 제안 섹션을 추가한다.
- 히어로 영역의 신뢰/효익 메시지를 보강한다.
- 기존 랜딩 톤에 맞춰 문구와 시각 강조를 정리한다.
- PDCA 산출물(`plan/design/tasks/report`)을 feature 경로에 생성/업데이트한다.

## Non-goals
- 대시보드 분석 로직/API/스토어 구조 변경.
- 인증/저장/공유 라우트 기능 변경.
- 전체 사이트 리브랜딩.

## Assumptions
- 서비스 주요 사용자(투자자/중개/PM)는 빠른 의사결정 근거를 중요하게 본다.
- 랜딩 전환 포인트는 검색 입력 전후 "신뢰성 + 시간 절감 + 실행 가능성" 전달이다.
- 기존 디자인 언어(blue/slate, card-based)를 유지하는 것이 일관성에 유리하다.

## Acceptance Criteria
- 랜딩에 신규 가치 제안 섹션이 추가된다.
- 히어로에서 신뢰/효익 메시지가 텍스트로 명확히 노출된다.
- 기존 섹션 흐름(히어로 -> 기능 -> 통계 -> ... -> CTA)을 깨지 않는다.
- 변경 파일 lint 검증을 통과한다.
- `npm run lint` 기준을 통과한다.

## Risks / Dependencies
- 문구 강화 과정에서 정보 과밀로 가독성이 떨어질 수 있다.
- 텍스트 길이 증가로 모바일 줄바꿈이 어색해질 수 있다.
- 랜딩 품질 향상과 실제 전환율 상승은 별도 측정이 필요하다.
