# KPI 정의서 (대표 보고 공통 기준)

## 목적
- 지점/담당자마다 다르게 쓰이는 KPI 용어를 통일해 대표 보고 시 오해를 줄인다.

## KPI 표준 정의

### 1) 신규 리드 수
- 정의: 보고 기간 내 신규 유입된 상담 리드 건수
- 계산식: `신규 생성 리드 총건수`
- 단위: `건`
- 데이터 출처: CRM/상담 관리 시스템
- 산정 주의:
  - 중복 리드(동일 연락처/동일 문의) 제거 규칙을 사전 합의

### 2) 유효 매물 수
- 정의: 보고 기간 말 기준 거래 가능한 상태의 활성 매물 수
- 계산식: `전체 매물 - 거래중지 - 만료 - 중복`
- 단위: `건`
- 데이터 출처: 매물 관리 시스템
- 산정 주의:
  - 상태값 기준일을 보고서 `asOfDate`와 맞출 것

### 3) 계약 전환율
- 정의: 유효 리드가 계약으로 전환된 비율
- 계산식: `(계약 완료 리드 수 / 유효 리드 수) * 100`
- 단위: `%`
- 데이터 출처: CRM + 계약 관리 시스템
- 산정 주의:
  - 분모(유효 리드 수)가 0인 경우 `N/A` 처리하고 코멘트 필수

### 4) 평균 처리 리드타임
- 정의: 리드 생성부터 계약 또는 종결까지 평균 소요일
- 계산식: `Σ(종결일 - 리드 생성일) / 종결 건수`
- 단위: `일`
- 데이터 출처: CRM/업무일지
- 산정 주의:
  - 미종결 건은 별도 버킷으로 관리(평균 왜곡 방지)

### 5) 민원/클레임 건수
- 정의: 보고 기간 내 접수된 고객 불만/클레임 수
- 계산식: `신규 접수 건수`
- 단위: `건`
- 데이터 출처: VOC/고객센터 시스템
- 산정 주의:
  - 동일 이슈 재접수 건의 집계 기준(건별/고객별) 사전 합의

## 품질 관리 규칙
- 모든 KPI는 `asOfDate`, `sourceSystem`, `owner`를 함께 표기한다.
- 산정 로직이 바뀌면 버전(`KPI Definition vX.Y`)을 보고서 메타정보에 기록한다.
- 지표 수치와 부록 원천 데이터가 불일치하면 보고서 배포를 보류한다.
